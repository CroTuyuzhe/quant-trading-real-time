# 实时趋势买卖信号系统

## 触发指令

在对话中输入以下指令触发：

- `/buy 6位股票代码` — 买入分析，例如 `/buy 600900`
- `/sell 6位股票代码` — 卖出分析，例如 `/sell 000001`

不区分大小写：`/BUY 300750`、`/Sell 000001` 均可。

## 核心逻辑

**只分析当天分钟数据，不看历史交易日。**

- 用户 `/buy`：检测上升趋势是否正在形成 → 是则买入，否则等待
- 用户 `/sell`：检测下跌趋势是否正在形成 → 是则卖出，否则等待

## 执行步骤

1. 从用户消息中提取 action（buy 或 sell）和 code（6位股票代码）
2. 确认是交易时间（9:30-11:30, 13:00-15:00，周一至周五），非交易时间提醒用户
3. 运行脚本开始循环监控：

```bash
python3 scripts/rt_signal.py --code CODE --action buy --interval 30 --max-minutes 30
```

脚本路径为 `scripts/rt_signal.py`（相对于本 skill 目录）

4. 每次循环输出一行摘要，出现 `__SIGNAL__:` 行时信号已触发
5. 出现 `__DONE__` 时脚本结束
6. 用中文总结给用户：信号、综合评分、当前价、推理链

## 信号类型

| 信号 | 含义 | 推理链示例 |
|------|------|-----------|
| 买入 | 上升趋势形成中 | 上升趋势加速形成 + 多头排列强化 |
| 卖出 | 下跌趋势形成中 | 下跌趋势加速 + 见顶信号 |
| 等待 | 趋势不明或方向不利 | 上涨加速中不宜卖 + 突破阶段 |

## 6维趋势因子

1. 价格趋势（短长期斜率方向）
2. 动量状态（三段斜率加减速）
3. 均线排列（MA5-MA20差值变化）
4. 量能趋势（近10分钟成交量斜率）
5. 反转信号（见顶/见底检测）
6. 突破阶段（蓄势/确认/失败）

## 数据源

- 主：新浪财经（行情+五档盘口+分钟K线）
- 兜底：腾讯财经 → akshare

## 注意事项

- 仅交易时间有效，非交易时间自动等待
- 只基于当天数据，不引入历史交易日
- 趋势分析需足够分钟数据点，开盘前30分钟信号可能不准
