# 实时趋势买卖信号系统

## 触发条件

当用户消息匹配以下模式时激活本技能：
- `buy` + 6位股票代码 — 用户想买入某只股票
- `sell` + 6位股票代码 — 用户想卖出某只股票

不区分大小写。示例：`buy 600900`、`sell 000001`、`buy 300750`。

## 核心逻辑

**只分析当天分钟数据，不看历史交易日。**

- **用户要 BUY**：检测上升趋势是否正在形成 → 是则买入，否则等待
- **用户要 SELL**：检测下跌趋势是否正在形成 → 是则卖出，否则等待

## 执行步骤

1. 从用户消息中提取 action（buy 或 sell）和 code（6位股票代码）
2. 确认是交易时间（9:30-11:30, 13:00-15:00，周一至周五），非交易时间提醒用户
3. 运行脚本开始循环监控：

```bash
python3 scripts/rt_signal.py --code CODE --action buy --interval 30 --max-minutes 30
```

脚本路径：`scripts/rt_signal.py`（相对于本 skill 目录）

4. 每次循环输出一行摘要：

```
[HH:MM:SS] #轮次 等待/买入/卖出  综合分  价格
        推理链
```

5. 出现 `__SIGNAL__:` 行时信号已触发，解析 JSON 获取详情
6. 出现 `__DONE__` 时脚本结束
7. 用中文总结给用户：信号、综合评分、当前价、推理链

## 调试模式

单次分析不循环，加 `--once`：

```bash
python3 scripts/rt_signal.py --code 600900 --action buy --once
```

## 输出解读

| 信号 | 含义 |
|------|------|
| 买入 | 上升趋势形成中，建议买入 |
| 卖出 | 下跌趋势形成中，建议尽快卖出 |
| 等待 | 趋势不明或方向不利，不宜操作 |

推理链示例：
- `上升趋势加速形成 + 多头排列强化` → 强买入信号
- `下跌趋势加速 + 见顶信号` → 强卖出信号
- `上升趋势加速，不宜卖出 + 突破阶段` → 趋势向好，等更好卖点

## 6维趋势因子

1. **价格趋势** — 短长期斜率方向，趋势是基础
2. **动量状态** — 三段斜率变化率，加速减速是拐点前兆
3. **均线排列** — MA5与MA20差值变化，多头空头排列强化或瓦解
4. **量能趋势** — 近10分钟成交量斜率，放量确认趋势
5. **反转信号** — 下跌减速加量回升等于底部，上涨减速加放量等于顶部
6. **突破阶段** — 距前高距离加成交量，蓄势确认失败

## 数据源

- 主：新浪财经（行情+五档盘口+分钟K线）
- 兜底：腾讯财经 → akshare

## 注意事项

- 仅交易时间有效，非交易时间自动等待
- 只基于当天数据，不引入历史交易日
- 趋势分析需足够分钟数据点，开盘前30分钟信号可能不准
- 综合分 0.72 以上触发强信号，0.62 以上且趋势因子支持触发中等信号
